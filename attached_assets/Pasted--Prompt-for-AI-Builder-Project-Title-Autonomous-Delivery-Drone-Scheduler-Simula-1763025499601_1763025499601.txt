**Project Title:** Autonomous Delivery Drone Scheduler (Simulation-Based)

**Goal:**
Build a **simulation** (no real drone hardware control) that models and visualizes how multiple autonomous delivery drones are scheduled and coordinated efficiently using **operating system scheduling principles** such as **priority scheduling, preemption, and synchronization**.

### üß© **Project Overview**

The system should simulate a fleet of drones performing delivery tasks (pickup ‚Üí delivery ‚Üí charging) under the control of a **central scheduler**.
Each **drone is modeled as a thread**, and **tasks are stored in a priority queue**.
Synchronization mechanisms (like mutex locks or semaphores) will be used to ensure that shared resources (like **charging stations** and **loading bays**) are accessed safely.

The aim is to **visualize and analyze how scheduling algorithms handle concurrent operations** in a real-world‚Äìinspired logistics scenario.

---

### üß† **Core Components**

1. **Drone Threads**

   * Each drone runs as a separate thread.
   * Attributes:

     * Drone ID
     * Speed (affects delivery time)
     * Battery level
     * Task queue (deliveries assigned)
   * Behavior:

     * Executes delivery tasks.
     * Requests charging when battery is low.
     * Waits if no task is available.

2. **Delivery Task Queue**

   * Implemented as a **priority queue** (higher priority = urgent delivery).
   * Each task contains:

     * Task ID
     * Source location
     * Destination location
     * Priority (1 = highest)
     * Estimated delivery time

3. **Central Scheduler**

   * Continuously monitors the task queue and drone statuses.
   * Assigns tasks to drones based on:

     * Drone availability
     * Task priority
     * Drone battery level
   * Handles **preemption** (urgent task interrupts current low-priority delivery).
   * Reassigns tasks if a drone becomes unavailable (low battery or charging).

4. **Synchronization (Concurrency Control)**

   * **Mutex locks / semaphores** ensure drones don‚Äôt simultaneously use shared resources.
   * Shared resources:

     * Charging stations (limited count)
     * Loading bays
   * Proper logging or visualization when resources are locked/unlocked.
